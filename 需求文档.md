需求文档
引言
SeekJob Helper 是一款智能辅助应用，旨在帮助信息技术（IT）与计算机科学专业学生求职。系统提供三大核心功能：基于 STAR 法则的简历优化、人工智能（AI）驱动的技术面试模拟，以及含 AI 自动评分的常见面试题练习题库。
术语表
SeekJob 系统：包含前端与后端组件的完整 Web 应用
用户：寻求面试准备的 IT / 计算机科学专业学生
简历优化器：分析并改进简历内容的功能组件
模拟面试官：AI 驱动的面试模拟功能组件
题库：面试题目的数据库及管理系统
STAR 法则：情境（Situation）、任务（Task）、行动（Action）、结果（Result）—— 一种结构化的经历描述方法
八股文：常见技术面试题及标准答题模板
AI 评分器：对照标准答案为用户作答评分的功能组件
功能需求
需求 1：简历优化
用户故事：作为求职学生，我希望上传简历文本并获取基于 STAR 法则的优化建议，从而提升简历质量，更清晰地展示个人经历。
验收标准
当用户通过输入界面提交简历文本时，SeekJob 系统应接收并存储该文本以进行后续处理
当简历优化器处理简历内容时，SeekJob 系统应基于 STAR 法则原理分析内容，并生成结构化优化建议
优化完成后，SeekJob 系统应以清晰格式展示原始内容及对应的优化建议（双语对照排版）
当用户查看优化结果时，SeekJob 系统应采用 Markdown 格式呈现建议，并配合适当的文本样式
若简历文本为空或仅包含空白字符，SeekJob 系统应拒绝提交并显示相应错误提示
需求 2：技术面试模拟
用户故事：作为求职学生，我希望参与含代码评估功能的 AI 模拟面试，在贴近真实的环境中练习技术面试。
验收标准
当用户启动模拟面试会话时，SeekJob 系统应创建新的对话上下文并显示聊天界面
当模拟面试官生成回复时，SeekJob 系统应采用 Markdown 格式呈现内容，包括代码语法高亮
当消息中包含代码块时，SeekJob 系统应根据指定编程语言应用语法高亮，提升可读性
当用户发送消息时，SeekJob 系统应将其追加至对话历史，并触发 AI 生成回复
当回复中包含数学公式时，SeekJob 系统应采用 KaTeX 格式渲染公式
需求 3：题库增删改查（CRUD）操作
用户故事：作为求职学生，我希望管理个人收藏的面试题目，构建个性化学习资源库。
验收标准
当用户创建新题目时，SeekJob 系统应将题目文本、标准答案及标签存储至题库
当用户请求题目列表时，SeekJob 系统应检索并展示所有题目及其元数据
当用户更新已有题目时，SeekJob 系统应修改存储的题目内容，同时保留其唯一标识
当用户删除题目时，SeekJob 系统应将其从题库中永久移除
若题目数据无效或不完整，SeekJob 系统应拒绝操作并返回验证错误提示
需求 4：随机题目练习
用户故事：作为求职学生，我希望从个人题库中随机抽取题目练习，以不可预测的方式检验知识掌握程度。
验收标准
当用户请求随机题目时，SeekJob 系统应采用均匀随机分布从题库中抽取一道题目
当随机题目展示时，SeekJob 系统应先仅显示题目文本，隐藏标准答案
若题库为空，SeekJob 系统应告知用户暂无可用题目
当用户选择查看答案时，SeekJob 系统应以适当格式展示标准答案
需求 5：AI 答题评分
用户故事：作为求职学生，我希望获取 AI 对个人作答的评分及与标准答案的对比分析，明确自身表现及改进方向。
验收标准
当用户提交题目作答时，SeekJob 系统应将用户答案与标准答案一同发送至 AI 评分器
当 AI 评分器处理作答时，SeekJob 系统应生成量化分数及详细分析报告
评分完成后，SeekJob 系统应展示分数、分析结果及与标准答案的对比
若用户作答为空，SeekJob 系统应拒绝提交并提示输入有效内容
当 AI 评分器生成分析报告时，SeekJob 系统应采用 Markdown 格式呈现反馈，并配合适当排版
需求 6：前后端通信
用户故事：作为系统组件，前端需与后端 API 进行安全通信，确保数据可靠传输。
验收标准
当前端发起 API 请求时，SeekJob 系统应采用与操作对应的 HTTP 方法（GET、POST、PUT、DELETE）
当后端接收到允许来源的请求时，SeekJob 系统应处理该请求并返回相应的跨域资源共享（CORS）头信息
当返回 API 响应时，SeekJob 系统应采用统一的 JSON 数据结构及规范的状态码
若请求因网络或服务器错误失败，SeekJob 系统应向前端返回描述性错误信息
当前端运行于 localhost:5173 时，SeekJob 系统应允许来自该来源的跨域请求
需求 7：数据持久化
用户故事：作为系统管理员，我希望题库数据在应用重启后得以保留，避免用户学习资料丢失。
验收标准
当 SeekJob 系统启动时，应初始化或连接至 SQLite 数据库以存储数据
当题目数据被创建或修改时，SeekJob 系统应立即将变更提交至数据库
当应用重启时，SeekJob 系统应从数据库中检索所有先前存储的题目
若数据库操作失败，SeekJob 系统应记录错误日志并返回相应的错误响应
当数据库模式初始化时，SeekJob 系统应创建所有必要的表，并设置适当的约束条件
需求 8：内容渲染
用户故事：作为求职学生，我希望看到格式规范的内容（包括 Markdown、代码及数学公式），确保技术信息清晰可读。
验收标准
当接收到 Markdown 内容时，SeekJob 系统应解析并转换为规范的 HTML 结构
当内容包含代码块时，SeekJob 系统应根据指定语言应用语法高亮
当内容包含 LaTeX 格式的数学公式时，SeekJob 系统应采用 KaTeX 进行渲染
当使用 GitHub 风格 Markdown 特性时，SeekJob 系统应支持表格、任务列表及删除线格式
当渲染内容时，SeekJob 系统应对 HTML 进行安全过滤，防止跨站脚本（XSS）漏洞
